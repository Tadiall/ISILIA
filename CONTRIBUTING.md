# CONTRIBUTING

# Contributing to ISILIA

First of all, thank you for considering contributing to ISILIA.

Whether you are a learner, educator, engineer, researcher, designer, writer, or simply someone who cares about education, your help is appreciated.

ISILIA is an open project built around a simple idea:

> Serious learners deserve serious guidance.

Our goal is not to collect the largest number of contributions.

Our goal is to maintain the highest possible quality.

Quality always comes before speed.

---

# Before You Contribute

Please read:

* README.md
* MANIFESTO.md
* VISION.md
* MISSION.md
* ROADMAP.md
* GOVERNANCE.md

Understanding the philosophy behind the project is important before proposing changes.

---

# What We Are Looking For

We welcome contributions that improve:

### Curriculum Quality

Examples:

* prerequisite corrections
* better learning sequences
* curriculum reviews
* missing topics
* improved project milestones

---

### Resource Quality

Examples:

* better books
* better courses
* better lectures
* better exercises
* updated documentation

A resource should not be recommended because it is popular.

It should be recommended because it is useful.

---

### Documentation

Examples:

* clearer explanations
* typo corrections
* formatting improvements
* missing information

---

### Platform Development

Examples:

* bug fixes
* performance improvements
* accessibility improvements
* user experience improvements

---

# What We Are NOT Looking For

Please do not submit contributions that:

* reduce academic rigor
* remove important prerequisites
* prioritize popularity over quality
* introduce unnecessary complexity
* add resources without justification
* promote personal products or courses

ISILIA is not an advertising platform.

---

# Contribution Principles

## Principle 1 — Clarity

Learning pathways should be easy to understand.

If a learner cannot understand the roadmap, the roadmap has failed.

---

## Principle 2 — Rigor

We prefer difficult truths over easy shortcuts.

A curriculum should prepare learners for mastery, not merely completion.

---

## Principle 3 — Practicality

Whenever possible:

* concepts should be applied
* projects should be meaningful
* learning should produce evidence

---

## Principle 4 — Accessibility

Whenever possible:

* prioritize free resources
* prioritize open resources
* avoid unnecessary barriers

---

# Resource Suggestions

When proposing a resource, explain:

1. What the resource teaches.
2. Why it is useful.
3. Which learners benefit most.
4. How it compares to existing resources.

Example:

```text
Resource:
MIT 6.006 Algorithms

Reason:
Excellent balance between theory and implementation.

Level:
Intermediate

Suggested Location:
Computer Science → Algorithms
```

---

# Curriculum Changes

Major curriculum changes require explanation.

Examples:

* adding a new subject
* removing a subject
* changing prerequisites
* reorganizing a learning path

Do not simply say:

```text
I think this is better.
```

Explain why.

Provide evidence.

Provide reasoning.

Provide educational justification.

---

# Pull Requests

Before opening a Pull Request:

1. Check existing Issues.
2. Check existing Discussions.
3. Explain your reasoning.
4. Keep changes focused.

Small, focused Pull Requests are easier to review than large ones.

---

# Discussions

Not every idea requires a Pull Request.

Use Discussions for:

* proposing ideas
* debating curriculum structure
* discussing educational philosophy
* requesting feedback

Use Issues for:

* bugs
* broken links
* specific improvements
* actionable tasks

---

# Review Process

Contributions may be:

* accepted
* revised
* postponed
* rejected

Rejection is not personal.

The goal is to protect the quality of the project.

Every contribution is evaluated according to:

* educational value
* clarity
* rigor
* usefulness
* alignment with the mission

---

# Community Expectations

We expect contributors to:

* remain respectful
* stay open to feedback
* discuss ideas professionally
* focus on evidence rather than authority

Strong disagreement is acceptable.

Personal attacks are not.

---

# Academic Humility

Nobody knows everything.

Every contributor, including maintainers and founders, can be wrong.

The purpose of discussion is not to win arguments.

The purpose is to improve the project.

---

# First Contributions

Good first contributions include:

* fixing typos
* improving explanations
* verifying links
* reviewing resource quality
* identifying missing prerequisites
* suggesting exercises

These contributions are valuable and help new contributors become familiar with the project.

---

# Final Reminder

ISILIA is a long-term project.

We are not trying to build content quickly.

We are trying to build guidance that remains useful for years.

If your contribution helps learners navigate knowledge more effectively, it is likely aligned with the mission.

Thank you for helping improve ISILIA.
